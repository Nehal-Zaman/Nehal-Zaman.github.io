# INTRODUCTION

Hello everyone. 

Join me in this blog post as we dive into the world of [Pilgrimage](https://app.hackthebox.com/machines/549), an approachable Linux box crafted by the talented [coopertim13](https://app.hackthebox.com/users/55851) over at [HackTheBox](https://app.hackthebox.com/home).

Our journey begins with a fascinating discovery â€“ the exposed `.git` repository, which sets the stage for our exploration of an intriguing vulnerability in `ImageMagick`. You'll witness how we transform this knowledge into an automated exploitation process.

But that's just the start of our adventure. As we progress, we'll uncover a privilege escalation opportunity through a vulnerability in the `binwalk` tool.

So, let's dive in and uncover the steps, the challenges, and the victories that make up the world of Pilgrimage.

# EXPLORING THE EXPOSED `.git`

![](/assets/images/writeups/pilgrimage/5.png)

We've come across an accessible `.git` folder, even though it returns a `403` status code instead of a `404`.

This can be potentially risky, as a malicious user could utilize the `.git` folder to retrieve the entire GitHub repository that holds the backend code of the web application.

In the `exec` function, a tool called `magick` is used to shrink images.

This tool is a type of software known as `ImageMagick`, and the version being utilized is `7.1.0-49 beta`.

A simple online search reveals that this specific version of `ImageMagick` contains a vulnerability referred to as `CVE-2022-44268`, which allows unauthorized access to files.

When a `PNG` image is passed to the `convert` command of the `magick` tool, it triggers a function called `ReadOnePNGImage`, which processes the image and reads text within it. Inside the code, there's a condition that checks for the presence of the keyword `profile` in the image's metadata. If this keyword is found, the code treats any accompanying text as a file name. By employing a function named `FileToBlob`, the tool then accesses and embeds the contents of that file into the image.

Further information about this vulnerability can be found [here](https://github.com/duc-nt/CVE-2022-44268-ImageMagick-Arbitrary-File-Read-PoC).
